# 해시 함수와 해시 코드 (Hash Function & Hash Code)

## 1. 해시(Hash)란 무엇인가?

**해시(Hash)**는 임의의 길이의 데이터를 고정된 길이의 데이터로 변환하는 과정 또는 그 결과물을 의미합니다. 데이터의 '지문(fingerprint)'이나 '요약본'과 같다고 생각할 수 있습니다.

- **해시 함수 (Hash Function):** 이 변환 과정을 수행하는 함수입니다.
- **해시 값/코드 (Hash Value/Code):** 해시 함수의 결과로 나오는 고정된 길이의 값입니다.

예를 들어, 아무리 긴 책의 내용이라도 해시 함수를 통과하면 항상 똑같은 길이의 짧은 문자열(해시 코드)로 변환됩니다.

## 2. 해시 함수의 주요 특징

모든 해시 함수는 다음과 같은 기본적인 특징을 가집니다.

- **결정론적 (Deterministic):** 동일한 입력값에 대해서는 항상 동일한 해시 값을 반환합니다.
- **빠른 계산 속도 (Fast Computation):** 입력값에 대한 해시 값을 계산하는 과정이 매우 빠릅니다.
- **고정된 출력 길이 (Fixed-Length Output):** 입력 데이터의 크기와 상관없이 항상 고정된 길이의 해시 값을 출력합니다.

## 3. 암호학적 해시 함수의 특징

단순 데이터 검색 등을 위한 일반 해시 함수와 달리, 보안 목적으로 사용되는 암호학적 해시 함수는 다음과 같은 추가적인 특징을 반드시 만족해야 합니다.

- **역상 저항성 (Pre-image Resistance):** 해시 값만으로는 원래의 입력값을 찾아내는 것이 계산상으로 거의 불가능합니다. (단방향성)
- **제2 역상 저항성 (Second Pre-image Resistance):** 특정 입력값과 동일한 해시 값을 갖는 또 다른 입력값을 찾아내는 것이 매우 어렵습니다.
- **충돌 저항성 (Collision Resistance):** 서로 다른 두 개의 입력값이 우연히 동일한 해시 값을 가질(충돌) 가능성이 매우 낮으며, 의도적으로 그런 쌍을 찾아내기 어렵습니다.

## 4. 해시 함수의 주요 용도

- **데이터 무결성 검사:** 파일을 다운로드할 때, 제공된 해시 값과 다운로드한 파일의 해시 값을 비교하여 파일이 중간에 변경되거나 손상되지 않았는지 확인합니다.
- **비밀번호 저장:** 사용자의 비밀번호를 직접 저장하는 대신, 해시 값으로 변환하여 저장합니다. 데이터베이스가 유출되더라도 원래의 비밀번호를 알기 어렵게 만듭니다.
- **자료 구조 (해시 테이블):** 데이터를 빠르게 저장하고 검색하기 위한 자료 구조(예: Python의 딕셔너리)에서 키-값 쌍을 효율적으로 관리하는 데 사용됩니다.
- **디지털 서명 및 블록체인:** 데이터의 위변조를 방지하고, 각 데이터 블록을 체인처럼 연결하는 데 핵심적인 역할을 합니다.

## 5. 대표적인 해시 알고리즘

| 알고리즘 | 출력 크기 (비트) | 보안 상태 | 주요 특징 |
| :--- | :--- | :--- | :--- |
| **MD5** | 128 | **취약함 (사용 금지)** | 충돌 취약점이 발견되어 보안용으로 절대 사용 불가 |
| **SHA-1** | 160 | **취약함 (사용 금지)** | 충돌 취약점이 발견되어 보안용으로 사용 권장 안 함 |
| **SHA-256** | 256 | **안전함 (현재 표준)** | 현재 가장 널리 사용되는 안전한 해시 알고리즘 |
| **SHA-512** | 512 | **안전함** | 64비트 시스템에서 더 빠른 성능을 보일 수 있음 |
| **SHA-3** | 가변 | **안전함 (차세대 표준)** | SHA-2와 다른 '스펀지 구조'를 사용하여 보안성 강화 |

### 요약

해시 함수는 데이터를 고유하고 간결한 '지문'으로 변환하는 강력한 도구입니다. 특히 보안과 관련된 시스템을 만들 때는 **충돌에 안전한 최신 암호학적 해시 함수(예: SHA-256 이상)를 사용하는 것이 매우 중요합니다.**
